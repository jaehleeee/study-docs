# 빈 생명주기 콜백
 * 스프링 빈은 객체 생성 후 의존관계 주입을 시작한다. 그리고 의존관계 주입이 끝난 다음에야 필요한 데이터 사용준비가 완료된다.
   * 데이터 사용준비 완료 시점을 알 수 있게 하기 위해, 스프링은 해당 시점에 대한 콜백 메서드 기능들을 제공한다.
 * 스프링 빈의 이벤트 라이프 사이클
   * 스프링 컨테이너 생성 -> 빈 생성 -> 의존관계 주입 -> 초기화 콜백 -> (애플리케이션 사용) -> 사멸전 콜백 -> 스프링 종료
