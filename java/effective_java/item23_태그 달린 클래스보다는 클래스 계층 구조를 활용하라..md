# 23. 태그 달린 클래스보다는 클래스 계층 구조를 활용하라.

## 핵심 정리
#### 태그달린 클래스란?
 * 필드중에 일부가 그 클래스의 타입을 나타내는 경우가 있다

#### 예시
```
class Figure {
  enum Shape { RECTANGLE, CIRCLE };

  final Shape shape; // 태그 필드

  // 사각형 일때만 사용하는 필드
  double length;
  double width;

  // 원 일때만 사용하는 필드
  double radius;
}
```


#### 태그달린 클래스의 단점
 * 불필요 코드 많다
 * 가독성 나쁘다
 * 메모리 많이 사용
 * 필드 final 선언하려면 불필요 필드까지 초기화해야한다.
 * 인스턴스 타입만으로는 현재 나타내는 의미를 알길이 없다.
