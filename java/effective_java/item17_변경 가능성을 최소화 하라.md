# 16. 변경 가능성을 최소화 하라
## 핵심 정리 (불변 클래스)
#### 불변 클래스는 가변 클래스보다 설계 및 구현하기 쉽고, 오류 여지가 적어 훨씬 안전하다
#### 불변 클래스 만드는 5가지 규칙
 * 객체 상태 변경 메서드(setter)를 제공하지 않는다.
 * 확장하지 못하게 한다. (public final class로 선언)
 * 모든 필드를 final로 선언
 * 모든 필드를 private으로 선언
 * 자신 외에는 내부의 가변 컴포넌트에 접근할 수 없도록 한다
